## FLAT vs SUBCLASS TRADEOFFS (vs ontology_v4.txt)
##
##   FLAT ADVANTAGES
##   - Simpler extraction: one node label per class; LLM emits a property string.
##   - Simpler merge: one merge key pattern per class type.
##   - Simpler Cypher for single-axis queries (no multi-label match needed).
##
##   FLAT DISADVANTAGES
##   - No structural guarantee on constraints: `category` valid only for domain=self;
##     `valence` valid only for channel=emotional. Must be enforced in pipeline / schema
##     validation, not by the graph structure itself.
##   - Indexing: domain/channel/subtype now need explicit property indexes in Neo4j
##     (CREATE INDEX FOR (n:CoreBelief) ON (n.domain)) to be as fast as label scans.
##   - Loss of label-based routing: extraction code can't branch on the Neo4j label;
##     must branch on the property value instead.
##
##   SAME AS V4 (unchanged)
##   - All edges, all edge semantics, all rules
##   - Situation.kind / temporality design
##   - distortionType enum (PatternReframe / CACTUS)
##   - technique enum (CACTUS 12)
##   - Homework.taskType
##   - AdaptiveResponse (content only, no modality)
##   - Goal targetsProblem Problem (direction)
##   - All deferred items (FormativeExperience, CopingStrategy, AdaptiveBelief)
